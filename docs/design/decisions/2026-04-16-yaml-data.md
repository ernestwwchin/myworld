# Decision: YAML for all game data

## Choice
All game content (maps, enemies, items, weapons, rules) defined in YAML files loaded via a Node/Express API.

## Why
- Human-readable and editable without touching JS
- Naturally moddable — content creators edit YAML, not code
- Maps can be hand-authored or generated and saved as YAML
- `js-yaml` parses reliably in both Node and browser

## Rejected alternatives
- **JSON** — no comments, harder to hand-author maps
- **Hardcoded JS objects** — tightly coupled, not moddable
- **SQLite/DB** — overkill for a single-player game with static content

## Trade-offs accepted
- YAML parsing adds a small startup cost
- Requires the Express server (can't run as pure static files without a build step)

## Status
Active. All `data/` folders use YAML. Schema validated at load time.
